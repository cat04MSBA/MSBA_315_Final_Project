# `data/` — How to populate this folder

The contents of `data/raw/`, `data/interim/`, and `data/processed/` are
**not committed** to git (the raw files alone are ~1.4 GB, well over GitHub's
limits). Only the folder structure is tracked, via `.gitkeep` placeholders.

To rebuild the working dataset from scratch, download the two raw sources
into `data/raw/` and run the build pipeline. Everything in `data/interim/`
and `data/processed/` is regenerated by code.

---

## 1. Expected layout

```
data/
├── raw/
│   ├── script_data/
│   │   ├── train.jsonl                 # MovieSum
│   │   ├── val.jsonl                   # MovieSum
│   │   └── test.jsonl                  # MovieSum
│   └── ratings_data/
│       └── IMDB TMDB Movie Metadata Big Dataset (1M).csv
├── interim/                            # built by pipeline (Phase 1 join)
└── processed/                          # built by pipeline (Phase 2 corpus)
```

---

## 2. Where to download the raw data

### MovieSum (screenplays) — `data/raw/script_data/`

- Source: Hugging Face dataset [`rohitsaxena/MovieSum`](https://huggingface.co/datasets/rohitsaxena/MovieSum)
- Reference: Saxena & Keller, ACL 2024
- ~50–100 MB total across `train.jsonl`, `val.jsonl`, `test.jsonl`
- License: research use

### IMDb-TMDB Movie Metadata (ratings) — `data/raw/ratings_data/`

- Source: Kaggle — *IMDB TMDB Movie Metadata Big Dataset (1M)*
- File needed (exact name, including spaces and parentheses):
  `IMDB TMDB Movie Metadata Big Dataset (1M).csv`
- ~1 GB
- License: see Kaggle dataset page (research / non-commercial use)

---

## 3. Building `interim/` and `processed/`

Once both raw sources are in place, run the pipeline from the project root:

```bash
python -m src.data.build_corpus              # builds data/processed/
python -m src.data.validate_processed_corpus # sanity-checks the build
```

`build_corpus` is idempotent and produces:

- `data/processed/films_joined.parquet` — master per-film table (1,713 × 41)
- `data/processed/screenplays_parsed.pkl` — parsed scene/dialogue structure

See `docs/DATA_NOTES.md` for the full column glossary and known caveats
(survivorship bias, year-density skew, etc.).
